2. Was kann schiefgehen?

2.1 Risikobewertung

Jede identifizierte Bedrohung wird mit der OWASP-Risk-Rating-Methodik bewertet. Eintrittswahrscheinlichkeit (Likelihood) und Auswirkung (Impact) sind jeweils der Durchschnitt von acht Faktoren, die von 0–9 bewertet werden; das Gesamtrisiko ergibt sich aus der 3×3-Matrix unten. Die Tabellen in diesem Abschnitt definieren, wie die in der Bedrohungstabelle (Abschnitt 2.3) gezeigten Bewertungen abgeleitet werden.

2.2 STRIDE-Kategorien

STRIDE strukturiert die Bedrohungsidentifikation entlang sechs Kategorien, die jeweils einer verletzten Sicherheitseigenschaft entsprechen. Die Leitfrage jeder Kategorie steuert die Analyse jeder Komponente und jedes Datenflusses im folgenden Abschnitt.

Kategorie Verletzte Eigenschaft Leitfrage
Spoofing (Identitätsfälschung) Authentizität Kann ein Angreifer eine andere Identität vortäuschen?
Tampering (Manipulation) Integrität Können Daten unbefugt verändert werden?
Repudiation (Abstreitbarkeit) Nichtabstreitbarkeit Kann ein Akteur eine Handlung glaubhaft abstreiten?
Information Disclosure (Informationsoffenlegung) Vertraulichkeit Kann unbefugt auf Daten zugegriffen werden?
Denial of Service (Dienstverweigerung) Verfügbarkeit Kann die Verfügbarkeit beeinträchtigt werden?
Elevation of Privilege (Rechteausweitung) Autorisierung Kann ein Angreifer unbefugte Rechte erlangen?

Die Risikobewertungen spiegeln den Zustand ohne angewendete Mitigationen wider. Die Spalte „Mögliche Mitigationen" wird in Phase 3 ausgefüllt. Ausnahme: Bei Bedrohungen, deren Schutzmaßnahme bereits im Code implementiert ist (T-11 und T-15, jeweils gekennzeichnet), ist die Bewertung als Restrisiko nach der bestehenden Kontrolle angegeben; das jeweilige Vor-Mitigation-Niveau ist im Bedrohungstext genannt.
